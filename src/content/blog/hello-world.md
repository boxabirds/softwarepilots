Welcome to the Software Pilots blog.

## What we will write about

This blog exists to explore a simple question: what does it mean to take responsibility for software you did not write line by line?

AI agents now generate working software from plain English. The code compiles. The tests pass. But who checked that the tests test the right thing? Who verified the security model? Who decided the architecture would hold under load?

That is what software pilotry is about.

## What is coming

We will be writing about:

- **Decomposition as a discipline** - why agents get stuck, and what a pilot does differently
- **Specification before generation** - the overlooked skill that separates good AI-assisted code from dangerous AI-assisted code
- **Verification patterns** - how to check work you did not write
- **The foundation course** - what we are teaching and why

Stay tuned.
